1. Introduction

Mass spectrometry of patient sera has become a central tool in cancer screening research because it captures the relative abundance of thousands of protein fragments in a single assay. Profiles that distinguish cancerous from healthy tissue carry biological information about the disease state, and a classifier trained on those profiles offers the promise of an early, noninvasive screening rule. The ARCENE benchmark, released for the NIPS 2003 Feature Selection Challenge and distributed by the UCI Machine Learning Repository, encapsulates this problem in a 200-sample, 10,000-feature dataset that has become a standard testbed for high-dimensional cancer classification.

The dataset embodies the high-dimensional regime in which the number of predictors greatly exceeds the number of observations, typically written as p >> n. In this regime classical multivariate methods break down. Ordinary least squares has infinitely many solutions, the within-class covariance matrix needed by linear discriminant analysis is singular, and any unregularized model with as many parameters as features will fit noise rather than signal. The methods designed for this regime fall into a few broad families. Regularization shrinks coefficients toward zero so the model has effectively fewer parameters. Dimension reduction projects the data onto a low-dimensional subspace before fitting. Embedded feature selection identifies a small subset of predictors with predictive value and discards the rest. Our analysis evaluates methods drawn from each of these families.

Our research question is which of these approaches gives the most accurate and interpretable classification of cancer versus normal samples on ARCENE. We compare five methods: principal component analysis followed by logistic regression, principal component analysis followed by linear discriminant analysis, bagging, a random forest with variable importance, and a guided regularized random forest. Because a missed cancer diagnosis carries greater clinical cost than a false alarm, we prioritize sensitivity (the fraction of true cancer cases correctly identified) when ranking the methods, while still reporting specificity and overall error to characterize the full performance profile.

2. Methodology

2.1 Data Description and Preprocessing

ARCENE, originally released for the NIPS 2003 Feature Selection Challenge and distributed by the UCI Machine Learning Repository, consists of 200 patient samples described by 10,000 continuous mass-spectrometry features and a binary response labeling each sample as cancer or normal. Roughly 3,000 of the predictors are deliberately included noise probes with no predictive power, which makes ARCENE a standard benchmark for high-dimensional feature selection. The class distribution (112 normal, 88 cancer) is moderately imbalanced but not extreme. We partition the data into an 80% training set of 160 observations and a 20% test set of 40 observations using a fixed random seed, and we drop columns with zero variance on the training matrix; the same columns are removed from the test matrix to keep the feature space aligned across the two sets.

2.2 Dimensionality Reduction (PCA)

Principal component analysis serves two purposes in our pipeline. First, it provides a low-dimensional view of the predictor space useful for visualization. Second, it is a necessary preprocessing step for linear discriminant analysis, which requires an invertible within-class covariance matrix; with p > n that matrix is singular on the raw predictors but becomes well-defined after projection onto a small enough set of leading components. We fit PCA on the standardized training matrix and project the test matrix onto the same components, so the test set never participates in the rotation. The number of components retained for downstream classifiers is fixed in advance by a non-test criterion: 20 components for the logistic regression step, chosen from inspection of the variance spectrum, and the smallest count needed to capture 95% of training variance for LDA. We report the resulting LDA component count in Section 3.1.

2.3 Classification Methods

PCA + Logistic Regression. Our first method reduces the predictor space to a small number of orthogonal directions of maximum variance, then fits a standard logistic regression in the reduced coordinates. The first stage uses principal component analysis on the training matrix; the second stage fits a binomial generalized linear model on the leading 20 components. PCA decouples the dimension reduction from the classification, and the choice of 20 components is fixed in advance from inspection of the variance spectrum rather than tuned against test accuracy.

PCA + Linear Discriminant Analysis. Linear discriminant analysis assumes that the two classes share a common covariance matrix and finds the linear combination of predictors that maximizes between-class separation relative to within-class spread. The shared-covariance assumption acts as a strong form of regularization at small sample sizes because it pools second-moment information across classes. Quadratic discriminant analysis, which would estimate a separate covariance matrix per class, doubles the number of covariance parameters and is unstable at our sample size, so we do not include it. We fit LDA on the principal components needed to capture 95% of training variance, with the resulting component count reported in Section 3.1.

Bagging. Bagging fits many decision trees on bootstrap resamples of the training data and aggregates their class votes by majority. The mechanism that drives the improvement over a single tree is variance reduction: averaging an ensemble of noisy but approximately unbiased predictors yields a more stable predictor than any single member. We implement bagging via the randomForest package with mtry set to the total number of predictors, so every split considers all available features and the procedure reduces to bagging in the textbook sense.

Random Forest with Variable Importance. A random forest extends bagging by drawing a fresh random subset of predictors at each split, which decorrelates the trees and reduces ensemble variance further than bagging alone. Setting importance = TRUE in the fit produces two per-feature importance scores: the mean decrease in classification accuracy when a feature is randomly permuted across out-of-bag samples, and the mean decrease in node impurity (Gini) attributable to splits on that feature. These scores serve both as an interpretive output describing which features drive the classifier and as input to the GRRF step that follows.

Guided Regularized Random Forest (GRRF). GRRF, introduced by Deng and Runger (2013), modifies the random forest splitting rule so that splits on features with low prior importance are penalized. The procedure first fits an unregularized regularized random forest (RRF) to obtain initial importance scores, then fits a guided RRF whose per-feature regularization coefficient is coef_j = (1 - gamma) + gamma * (importance_j / max(importance)), where larger gamma penalizes low-importance features more aggressively. We fix gamma at 0.5, the midpoint recommended by Deng and Runger, before running any model on the test set; the resulting selected feature set is reported as a candidate biomarker shortlist.

2.4 Model Evaluation

We evaluate every model on the held-out 20% test set, with Class 1 (cancer) as the positive class. From each model’s confusion matrix we compute three metrics: sensitivity, defined as TP / (TP + FN), the fraction of true cancer cases the model correctly identifies; specificity, defined as TN / (TN + FP), the fraction of true normal cases the model correctly classifies; and the overall test error rate, defined as (FP + FN) / N. We rank methods primarily by sensitivity because a false negative, a cancer patient classified as normal, carries materially greater clinical cost than a false positive, which typically only triggers a confirmatory test. For the LDA model we additionally report the area under the receiver operating characteristic curve (AUC), which is the probability that a randomly chosen cancer sample receives a higher predicted posterior than a randomly chosen normal sample, and which summarizes classifier performance across all decision thresholds.

On the test set, PCA + logistic regression with 20 components achieves sensitivity 0.923, specificity 0.741, and an overall error rate of 0.200. The result establishes a competitive linear baseline against which the more flexible methods below are measured.

On the test set, PCA + LDA with 130 components achieves sensitivity 1.000, specificity 0.852, and an overall error rate of 0.100. Every cancer case in the test split is correctly identified, and the misclassifications all fall on the normal side. The shared-covariance assumption pays off here: with 130 LDA coefficients to estimate from 160 training observations, pooling the within-class second moments into a single matrix produces a more stable discriminant direction than a class-specific estimator could.

On the test set, bagging achieves sensitivity 0.692, specificity 0.815, and error 0.225. The lower sensitivity relative to PCA + LDA reflects the difficulty trees face on this dataset: each split must choose a feature from 9940 noisy candidates, and bootstrap resampling alone is not enough to suppress the resulting per-tree variance even after averaging the ensemble. 0 18 0

On the test set, GRRF reduces the active feature set from 9940 candidates to 13 and achieves sensitivity 1.000, specificity 0.667, and error 0.225. The classifier identifies every cancer case in the test split but accepts a higher false-positive rate than PCA + LDA in exchange for the dramatic reduction in feature count. The compact selected set is the practical contribution of GRRF on this dataset: it produces a candidate biomarker shortlist that is small enough for follow-up biological validation in a wet-lab setting, where validating 9940 features would be infeasible but validating 13 is routine.

4. Discussion

PCA + LDA is the strongest classifier in our comparison. It achieves perfect test-set sensitivity (1.000), the highest specificity of any method (0.852), and the lowest overall error rate (0.100), and it dominates the alternatives on the primary metric we set in advance. GRRF also reaches perfect sensitivity (1.000) but accepts a substantially lower specificity (0.667) in exchange for a much smaller selected feature set. PCA + logistic regression and the random forest land in the middle of the comparison with identical test-set metrics in this split, while bagging trails the other four. The pattern points to two qualitatively different strong solutions in the high-dimensional regime: a pooled-covariance discriminant analysis on a low-dimensional projection of the predictors, and an ensemble of trees with explicit feature selection.

In a clinical screening context the metric that matters is sensitivity, because the cost of a missed cancer diagnosis dwarfs the cost of a false alarm that triggers a confirmatory test. By that criterion PCA + LDA is our primary recommendation: it correctly identifies every cancer case in the test set and does so with the highest specificity of any method that achieves perfect sensitivity. GRRF is positioned as a secondary tool with a different value proposition. Its 13-feature shortlist (out of 9940 candidates after zero-variance removal) is small enough for follow-up biological investigation in a laboratory setting, and the overlap between its selected features and the high-importance predictors flagged by the random forest can guide hypothesis generation about which protein abundances actually drive the cancer signal.

Several limitations qualify these conclusions. The reported metrics come from a single 80/20 train/test split with only 13 cancer cases in the test set, which leaves substantial sampling uncertainty around every point estimate; a one-case change in the test split can move sensitivity by several percentage points. We did not cross-validate the comparison, so we cannot put confidence intervals on the differences between methods. The PCA preprocessing step is unsupervised, which means the leading components maximize total predictor variance rather than between-class separation, and a supervised dimension reduction such as partial least squares might give the LDA pipeline more discriminative directions for the same component budget. The GRRF results depend on both the gamma parameter and the random seed; we fixed gamma at 0.5 in advance from the Deng and Runger recommendation, but we did not characterize how the selected feature count or test metrics vary across other reasonable choices. Finally, the dataset includes roughly 3,000 intentional noise probes whose biological identity is undefined, so the variable-importance and GRRF feature lists describe predictive value within ARCENE but not necessarily biological relevance in a real proteomic assay.

The natural next steps follow directly from the limitations above. Repeated stratified train/test splits or k-fold cross-validation would replace each point estimate in the comparison table with a sampling distribution and would let us put a confidence interval on the gap between PCA + LDA and the next best method. A supervised dimension reduction (partial least squares discriminant analysis is the obvious candidate) is worth comparing against PCA as the front end to LDA, because supervised projections often need fewer components for the same discriminative power. A stability analysis of the GRRF feature set across seeds and gamma values would reveal which of the 13 selected features are robustly chosen and which are artifacts of a single run. External validation on an independent cancer cohort, if one were available with comparable mass-spectrometry features, would test whether the LDA discriminant generalizes outside ARCENE. Any biomarker shortlist derived from these analyses, including the GRRF feature set, requires laboratory validation against an orthogonal measurement before it could inform clinical practice.
